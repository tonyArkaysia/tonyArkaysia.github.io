DeliOS Architecture Overview

How Nuthin' Left Deli runs on a scalable Stripe-powered platform: fast in-store checkout today, and multi-location, AI-ready analytics tomorrow.

One core platform; many delis, retailers, and future verticals.

System at a Glance

DeliOS is a Stripe-based retail platform. The Angular PWA is the face, the Railway backend is the brain, Stripe handles money flow, and Postgres/GCP store the data for future AI and BI.

Key Design Principles

  • Single platform serving many merchants (multi-tenant SaaS).
  • Stripe first: Stripe Terminal + web payments for unified in-store/online data.
  • API-first backend so web, PWA, future native apps all share the same brain.
  • AI-ready data model: detailed orders/payments stored in Postgres for future BI.

Value by Role

Nuthin' Left Deli is the first rollout. DeliOS is designed for many shops, without rebuilding payment plumbing later.

For the Deli Owner
  • Fast, reliable checkout with Stripe Terminal hardware.
  • Owner dashboard for sales, top items, staff performance.
  • Works on any device: Android/iOS phone, tablets, office desktop.
For A_Labs / Platform
  • Stripe Connect-ready: easily onboard new merchants into the same platform.
  • Rich data stream for AI, forecasting, and BI consulting.
  • Clear separation of concerns: UI (Angular), API (Railway), money (Stripe), storage (Postgres/GCP).

Enterprise-ready from store #1

Why Stripe for DeliOS (vs Square)

Stripe is chosen not just for today's deli payments, but for DeliOS as a SaaS platform. It gives enterprise-grade capabilities from day one.

Stripe ‚Äì Platform Advantages

  • Unified online + in-person payments under one customer/payment model.
  • Stripe Connect for multi-merchant onboarding and revenue models.
  • Deep, granular events for AI and BI pipelines (webhooks, ledgers).
  • Supports subscriptions, usage-based billing, and future SaaS plans.
  • Clean developer ecosystem for evolving DeliOS across verticals.

Trade-offs & Mitigations

  • More engineering effort than Square at setup ‚Üí solved by a clear Stripe integration layer.
  • Less visually familiar than Square terminal ‚Üí mitigated by training and consistent UI.
  • Onboarding feels more ‚Äúenterprise‚Äù ‚Üí framed as an upgrade, not a hurdle.

Scalability Story

DeliOS starts with one deli, but the architecture is built as if store #10 and #50 already exist:

  • Multi-tenant Postgres: merchants, locations, and users isolated by IDs and permissions.
  • Stripe Connect (future): onboard each deli as a connected account.
  • Event-driven integrations: Stripe webhooks feed analytics and AI models.
  • Horizontal API scaling: Railway services can be scaled out behind a load balancer.
  • Front-end independence: Angular PWA now, native/hybrid apps later, all sharing the same API.
